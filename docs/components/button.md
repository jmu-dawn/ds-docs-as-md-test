## Overview

Button is a pill-shaped action control offering four hierarchy styles (Primary, Secondary, Ghost, Elevated), an optional danger treatment, and two sizes (Default, small). It exposes independent optional leading and trailing icon slots — each renders a swappable instance defaulting to the Plus icon — and decomposes Figma’s four-value State axis into a single persistent `isDisabled` prop, since Hover and Pressed are transient states the platform handles at runtime. Only the Size axis changes dimensions (padding and text style scale down at small); Hierarchy, Danger, and State drive color exclusively, via per-hierarchy/danger token families across the Default/Hover/Pressed/Disabled interaction states. As a screen-reader target it is a single focus stop whose visible label merges into the accessible name, with decorative icons hidden and Disabled the only state that alters accessibility semantics.

**Variant axes.** State (Default*, Hover, Pressed, Disabled); Hierarchy (Primary*, Secondary, Ghost, Elevated); Size (Default*, small); Danger (False*, True). _(\* = default)_

## Structure

_Confidence: high._

> Button has one dimensional axis (Size: Default / small). The State (Default / Hover / Pressed / Disabled), Hierarchy (Primary / Secondary / Ghost / Elevated) and Danger (False / True) axes are visual-only: across all of them the layout tree, padding, spacing, corner radius and stroke weight are byte-for-byte identical (verified via crossVariant.axisDiffs and stateComparison), so they carry no dimensional section. Padding and spacing scale with size through the padding/_ and gap/_ token families (verticalPadding 12->8, horizontalPadding 24->16, itemSpacing constant 8). The container is HUG on both axes, so its rendered height (48 at Default, 36 at small) is a product of vertical padding plus text line-height, not a fixed constraint. cornerRadius is 999 in every variant — an effective pill that always clamps to half the rendered height. The two Plus icon instances are referenced library components gated by the Leading icon / Trailing icon booleans (hidden in the default variant); only their hosting size is documented here — see the Plus icon spec for internals. Known gap: strokeWeight resolves to 1 on the container in every variant, but this \_base.json (plugin 2.5.0) carries no strokes / strokePaintToken data, so it cannot be confirmed that a border is actually painted (vs. a stroke weight set with no paint). No borderWidth/borderAlign row is emitted; if hierarchies such as Secondary/Ghost rely on a visible border, re-extract with stroke-paint capture to confirm.

## Voice / Screen reader

_Confidence: high._

Button is a simple component with a single focus stop. The visible label text ("Regular button") merges into the button's accessible name, so it is never a separate focus stop. Leading and trailing icons are decorative and off by default; when shown they convey no independent meaning and must be hidden from screen readers (accessibilityElementsHidden / contentDescription = null / aria-hidden) so the label is never read twice. The accessible name should describe the action (e.g. "Submit"), not the appearance. Disabled is the only state that changes accessibility semantics: the button is removed from the focus order and exposes the disabled trait/attribute. Hover and Pressed are transient, platform-handled visual states with no distinct screen reader semantics, so they are not documented as separate states. Hierarchy, Size, and Danger are purely visual variant axes and do not affect screen reader behavior. Do not reorder focus across states; keep the stop index stable so user gestures remain predictable. Do not stuff the accessible name with extra hints or state; each belongs on its dedicated property. iOS: prefer a native Button / .isButton trait. Android: prefer a native Button with Role.Button. Web: prefer native `<button>` over role="button".
