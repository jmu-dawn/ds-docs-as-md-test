The design system is a shared language for building our products. It provides
foundations (color, typography, spacing) and a library of components so teams
can ship consistent, accessible interfaces faster.

## Principles

1. **Consistent** — the same problem is solved the same way everywhere.
2. **Accessible** — everything meets WCAG AA by default.
3. **Composable** — small pieces combine into larger patterns.

## Who it's for

Designers and engineers building product surfaces. Designers reference the
foundations and component specs; engineers consume the published tokens and
component library.
